Samurai
Dashboard.

Replatforming a Gulf retail distribution operation from a brittle Vue codebase to a fast, type-safe React Admin stack — with the lights on.

Year2024 — 2025
RoleLead Engineer
SectorRetail / Logistics
StatusShipped

Problem

// 01

A major Gulf retail distribution operation was running its day-to-day on a Vue 2 dashboard that had grown organically over years. Multiple teams had bolted on features. The UI was inconsistent. Onboarding new engineers took weeks. Page loads were slow on the heaviest CRUD screens.

Leadership wanted a modern stack but couldn't afford a freeze — the dashboard runs the business in real time, with delivery teams, ops, and admins logged in throughout the day. Any rewrite had to ship incrementally, in production, alongside the legacy app.

"Migrate without freezing. Keep shipping. Don't break the trucks on the road."

Approach

// 02
  • 01
    Audit & mappingCatalogued every Vue route, identified the highest-traffic and highest-debt modules, and ranked them by ROI for migration.
  • 02
    React Admin foundationStood up React Admin with a shared TypeScript types layer that both old and new code could consume, backed by the existing API.
  • 03
    Strangler-fig migrationPorted one module at a time. Old and new shipped side-by-side behind a switch — users hit the new UI, ops could fall back instantly.
  • 04
    UI/UX overhaulLayered Material UI consistently across migrated screens. Killed dead components. Tightened information density without losing legibility.
  • 05
    Live mappingAdded Google Maps integration for delivery tracking — live truck positions, route history, geofence alerts.

Stack

// 03

Outcome

// 04

Migration is ongoing — the project has shipped continuously since early 2024 with zero downtime windows for end users. Every migrated module has lower TTI than its Vue predecessor and noticeably faster turnaround on new feature requests.
