Go
Boon.

A hiring pipeline platform for small businesses — sourcing, screening, scheduling, and offers stitched into a single tool, with a mobile app for candidates and an admin panel for HR.

Year2019 — 2020
RoleSr. Full-Stack
SectorHR Tech
SurfaceMobile + Web

Problem

// 01

Small businesses were drowning in manual hiring. Multiple job boards, inboxes full of resumes, scheduling interviews via email chains, and offers pasted into Google Docs. Time-to-hire stretched into weeks for roles that should have closed in days.

The brief: a single platform that took a vacancy from "we need to hire someone" to "they signed the offer" without ever leaving the tool.

"Stop the hiring funnel from leaking through six different apps."

Approach

// 02
  • 01
    Pipeline-first data modelModeled vacancies as a state machine: posted → sourced → screened → scheduled → offered → hired. Every transition recorded.
  • 02
    Rails admin for HRA clean React + Rails admin panel for HR teams to manage the pipeline, with role-based permissions and team-level views.
  • 03
    React Native for candidatesA candidate-facing mobile app for application, scheduling, and offer review — with push notifications at every state change.
  • 04
    Automated screeningRules-based candidate screening to auto-advance or auto-reject based on configurable criteria, freeing HR to focus on judgment calls.

Stack

Outcome

// 04

End-to-end hiring funnel managed from one tool. HR teams could see every candidate's exact state, candidates got real-time updates instead of weeks of silence, and time-to-hire dropped meaningfully across the early-stage clients we deployed for.
