Run
Buddy.

A two-sided mobile marketplace pairing pet owners with vetted runners — with auth, Stripe payments, in-app chat, camera verification, and live maps.

Year2020 — 2021
RoleSr. Full-Stack
SectorMarketplace
PlatformsiOS · Android

Problem

// 01

Pet owners wanted scheduled workouts with vetted runners, but existing solutions were a hodgepodge of WhatsApp threads and PayPal links. Scheduling was painful, payments were off-platform, and there was no way for either side to verify the run actually happened the way it was supposed to.

The brief: build a real two-sided marketplace — sign up, verify, schedule, pay, run, review — that lived inside a single mobile app on both iOS and Android.

"Make booking a runner as easy as booking an Uber. Make trust come baked in."

Approach

// 02
  • 01
    One codebase, two appsReact Native for iOS & Android in a single TypeScript codebase, with Redux managing cross-cutting state.
  • 02
    Rails API + PostgresModeled users, runners, schedules, runs, payments, and reviews on a Rails backend with a normalized PostgreSQL schema.
  • 03
    Stripe for paymentsStripe Connect for split payouts — platform takes a cut, runner gets the rest, all on rails.
  • 04
    In-app chat & cameraBuilt real-time chat for owner ↔ runner coordination, plus camera flows for run-completion proof.
  • 05
    Live maps + geofencingGoogle Maps for matching, route preview, and post-run summaries with run path overlay.

Outcome

// 04

Shipped to App Store and Play Store with the full marketplace flow live: signup & verification, schedule, pay, run, review. Real-time match notifications, Stripe-backed split payouts, and proof-of-run via camera.
