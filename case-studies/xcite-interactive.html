Xcite
Interactive.

Web-based interactive games for live stadium audiences — thousands of fans playing simultaneously, synced sub-second to the jumbotron above the field.

Year2017 — 2018
RoleFront End Engineer
SectorLive Events
ScaleThousands concurrent

Problem

// 01

Stadium operators wanted fans in the stands to participate in live jumbotron experiences — vote, react, play games, get featured on the big screen — but every existing solution had unacceptable lag for live audience interaction.

The product needed to feel instant from a phone in the stands to the screen 200 meters away, and it had to scale to thousands of concurrent players without falling over mid-game.

"Sub-second from phone to jumbotron. Or it isn't worth shipping."

Approach

// 02
  • 01
    Web-based, no installBuilt on Angular + TypeScript so fans could join via a QR code on the screen — no app install, no friction.
  • 02
    Firebase Realtime + FirestoreUsed Firebase Realtime DB for sub-second event propagation and Firestore for persistent state, with careful attention to read/write fan-out.
  • 03
    Designed for fan-outArchitected the data model and rules so a single host action could ripple to thousands of clients without blowing the per-client read budget.
  • 04
    Jumbotron viewBuilt a separate operator/jumbotron view that aggregated fan input in real time and rendered to the big screen with the same backend.

Stack

// 03
Angular
TypeScript
Firebase
Firestore

Realtime DB
Node.js

Outcome

// 04

Deployed at live stadium events with thousands of fans playing concurrently, sub-second sync to the jumbotron above the field. The product became the foundation of further interactive live-event work at the studio.
